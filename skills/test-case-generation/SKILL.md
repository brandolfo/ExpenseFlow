# Test Case Generation Skill

Use this skill when defining tests for ExpenseFlow.

## When to use
Use when the task involves:
- Acceptance criteria
- Unit tests
- Integration tests
- Edge cases
- Validation
- Categorization correctness
- AI failure handling

## Test categories
Consider:
- Happy path
- Invalid file
- Missing columns
- Invalid dates
- Invalid amounts
- Duplicate transactions
- Negative amounts
- Refunds
- Installments
- Category conflicts
- No matching rule
- Low AI confidence
- Invalid AI output
- Total mismatch
- Empty file
- Very large file

## Output
Return:
- Test scenario
- Input
- Expected behavior
- Priority
- Automation level
- Notes
