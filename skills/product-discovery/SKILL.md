# Product Discovery Skill

Use this skill when defining or refining ExpenseFlow's product direction.

## When to use
Use when the task involves:
- Target users
- Pain points
- MVP definition
- Product scope
- User stories
- Acceptance criteria
- Out-of-scope decisions

## Steps
1. Identify the target user.
2. Describe the current workflow.
3. Identify the painful or repetitive steps.
4. Define the desired outcome.
5. Separate must-have from nice-to-have.
6. Identify explicit non-goals.
7. List assumptions.
8. Define validation questions.

## Output
Return:
- Target user
- Problem
- Current workflow
- Pain points
- Desired outcome
- MVP candidate
- Out of scope
- Assumptions
- Open questions
